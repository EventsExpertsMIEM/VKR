REST API (need to complete)

Далее идет описание REST API.

Основным форматом является json (но не всегда), для чего необходимо указывать в заголовках application/json. В случае отсутсвия данного заголовка или же некорретно составленного тела будет возвращена ошибка с http кодом 415.

Ниже приведены описания всех возможных ответов сервера:

200 - основной ответ в случае успешного выполнения запроса.
201 - ответ при успешном создании.

400 - ответ при неправильных ключах json.
401 - ответ незалогиненому пользователю.
403 - ответ при отсутствии доступа к выполняемому действию.
404 - ответ при отсутствии запрашиваемого пути или запрашиваемых данных.
405 - ответ при отсутствии метода у данного пути.
409 - ответ при конфликте состояния.
415 - ответ при некорректном json.
422 - ответ при некорретных поданых данных.

Accounts

Ручки, связанные с аккаунтами.

  • Авторизация пользователя в системе.
  • Принимает
    • email - почта пользователя
    • password - пароль пользователя

    • description

    • surname - фамилия пользователя

  • 409 - пользователь уже авторизован

  • 415 - прислан не json, или же он составлен неверно
  • 422 - неверный пароль
/logout - [GET/POST] - LOGIN REQUIRED
  • Логаут пользователя из системы.
  • 200 OK
    • description
  • 401 - не залогинен

  • Регистрация пользователя в системе.

    • surname - фамилия пользователя

    • description

    • country - страна текущего пользователя

Events

Ручки, связанные с мероприятиями.

  • Принимает

    • location - место проведения мероприятия
    • site_link - ссылка на собственный сайт мероприятия
    • additional_info - дополнительная информация мероприятия
  • 201 CREATED
    • description - id созданного мероприятия
  • 400 - некорректные ключи json
  • 401 - не залогинен
  • 404 - мероприятие с таким id не найдено
  • 415 - прислан не json, или же он составлен неверно
  • 422 - некорректные данные (скорее всего даты и время)

Reports

Ручки, связанные с докладами.

Все ручки данной категории имеют префикс /event (т.е. первая ручка / является /event/).

/[id]/report - [POST] - LOGIN REQUIRED
  • Загрузка доклада на мероприятие
  • Принимает
    • [id] - id мероприятия
    • multipart-form-data
      • file - файл с докладом (pdf, word/x, ppt/x)
  • 200 OK
    • description
  • 401 - не залогинен
  • 403 - отсутсвует файл
  • 404 - мероприятие не найдено
  • 413 - размер файла превышает допустимый
  • 415 - неподдерживаемый формат файла
  • 424 - пользователь не присоеденился к мероприятию
/report/all - [GET] - LOGIN REQUIRED + RIGHTS REQUIRED
  • Получает список всех загруженных докладов для всех мероприятий, доступна
  • 200 OK
    • Список
    • event_id - id мероприятия
    • user_id - id ползьвателя (докладчика)
    • report_id - id доклада
    • name - имя файла доклада
    • last_update - дата последнего изменения
    • status - статус доклада
/report/[id] - [GET]
  • Получить файл доклада по id
  • Принимает
    • [id] - id доклада
  • 200 OK
    • filename - файл доклада как attachment
  • 404 - доклад с заданным id не найден
/[id]/report - [GET] LOGIN REQUIRED
  • Получить файл доклада текущего пользователя для заданного мероприятия
  • Принимает
    • [id] - id мероприятия
  • 200 OK
    • filename - файл доклада как attachment
  • 404 - мероприятие с заданным id не найден или пользователь не загружал доклад
  • 424 - пользователь не присоеденился к мероприятию
/[id]/report/info - [GET] LOGIN REQUIRED
  • Получить информацию о докладе текущего пользователя на заданное мероприятие
  • Принимает
    • [id] - id мероприятия
  • 200 OK - JSON
    • report_id - id доклада
    • report_name - имя файла с докладом
    • last_updated - дата последнего изменения
    • status - статус доклада
  • 404 - мероприятие с заданным id не найден или пользователь не загружал доклад
  • 424 - пользователь не присоеденился к мероприятию
/[id]/report/all - [GET]
