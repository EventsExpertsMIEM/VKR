Events Service Page

Сервис мероприятий конгрессной деятельности

Это страница репозитория с кодом сервера Web-сервиса для организации и поддержки конгрессной деятельности - для автоматизации и упрощения взаимодействия "участник-организатор-исполнитель".

Основные функции сервиса:

  • Отслеживание мероприятий
  • Создание мероприятий и их настройка
  • Отслеживание статуса подготовки мероприятий и модерацией участников
  • Регистрация на мероприятия и получение информационной поддержки

Repo

Как работать в репозитории:

  • Создавать под фичи и задачи отдельные ветки
  • После завершения работы создавать ПР с указанием меня для проверки
  • ПР просьба оформлять грамотно
  • ПР делаются в ветку master

Структура репозитория:

  • В app лежит код самого бекенда
    • api - REST API
    • db - структура БД и инициализация
    • logic - внутрення логика
    • static - статические файлы для страниц
    • templates - шаблоны страниц для jinja2
    • web - urlы
    • __init__ - инициализация flask и модулей
    • congig - конфиг для всего сервера
    • errors - обработка ошибок
    • logging_config - конфиг логгеров сервера
    • mails - почта
  • main - запуск сервера
  • requirements.txt - необходимые pip пакеты
  • setup.sh - шаблон файла инициализации конфига в переменные окружения (shell)
  • setup.bat - шаблон файла инициализации конфига в переменные окружения (win)

What we use

Основное:

  • Язык программирования сервера - Python3
  • СУБД - PostgreSQL
  • ORM для БД - SQLAlchemy
  • Web-framework - Flask
  • Web-интерфейс - Bootstrap4

pip модули:

  • Хэширование паролей на сервере - bcrypt
  • Авторизация - Flask-Login
  • Работа с эл. почтой - Flask-Mail
  • WSGI coroutines - gevent
  • UID генерация - nanoid
  • Подключение к БД (PostgreSQL) - pg8000 и psycopg2-binary
  • Управление реквестами - requests
